Em Programming Language Specification

Em is a systems programming language with a C-style syntax that emphasizes clean, simple, and minimal features with high performance. This document describes the complete syntax and semantics of the Em programming language.

1 Introduction

Em is a systems programming language designed for simplicity and control over low-level details. The compiler, known as emc, generates efficient machine code through LLVM. Em supports modern programming constructs while maintaining minimal syntax overhead.

1.1 Project Information

2 Usage and Compilation

2.1 Basic Usage

To compile an Em source file, use the Em compiler emc: 

emc program.em

The source file must have a .em file extension. The compiler will produce an executable with the same name as the source file (minus the extension).

2.2 Compiling Multiple Files

To compile multiple Em source files together: 

emc file1.em file2.em file3.em

All specified files will be compiled and linked together into a single executable.

2.3 Header Files (.emh)

Em uses header files with the .emh extension to provide declarations and function prototypes. Header files are included in source files using the #include preprocessor directive. Associated compiled header libraries with .bc extension are automatically linked during compilation when headers are included. This allows code to be organized into reusable libraries and modules.

2.4 Compiler Flags

Compiler flags are specified after the list of files to compile. A flag starts with a hyphen (-): 

emc file1.em file2.em [flags]
Flag Description
-pout Prints the parser output (AST structure)
-llout Prints the generated LLVM IR
-ll Generates a .ll file (LLVM IR) instead of an executable
-asm Generates a .s file (Assembly) instead of an executable
-cpu <type> Specifies the target CPU type. Must be followed by a CPU name (see section 2.5)
-o <filename> Names the output file. Must be followed by the desired filename
-benchmark Prints compilation performance metrics

Example: 

emc prog.em -asm -cpu x86-64 -o prog.s

This command compiles prog.em, generates assembly output for x86-64, and saves it to prog.s.

2.5 CPU Targets

The following CPU types can be specified with the -cpu flag: 

x86-64
cortex-m3
cortex-m4
cortex-m7
cortex-a7
cortex-a53
cortex-a72
cortex-a76
cortex-a78
cortex-x1
apple-m1
apple-m2
neoverse-n1
neoverse-v1
neoverse-n2

If no target CPU is specified, the compiler automatically detects the host machine's CPU type.

3 Hello World Example

Here is a simple Hello World program in Em. Note that Em does not include built-in I/O functions; this example demonstrates the basic structure of an Em program with a main function: 

#include <print.emh>

int main()
{
    print("hello world!", 12);
    return 0;
}

This program defines a main function that takes no parameters and returns 0. To use I/O operations, you would need to use included header files from the standard library.

4 Lexical Structure

4.1 Tokens

The lexical structure of Em is composed of the following token types:

Token Type Examples Description
Identifier x, foo_bar, var123 Names for variables, functions, and other identifiers
Keyword if, while, return Reserved words in the language
Data Type int, float, bool Type identifiers
Numeric Literal 42, 3.14, 0xFF Integer and floating-point constants
Character Literal 'A', '0' Single character in single quotes
String Literal "hello", "world\n" Sequences of characters in double quotes
Boolean Literal true, false Boolean constants
Separator , Separates items in lists
Delimiter ; Terminates statements
Brackets { } ( ) [ ] Group code and expressions
Operators +, -, *, /, etc. Perform operations on values

4.2 Identifiers

Identifiers are names for variables, functions, and other program elements. An identifier:

  • Must start with a letter (a-z, A-Z) or underscore (_)
  • Can contain letters, digits (0-9), and underscores
  • Is case-sensitive
  • Cannot be a reserved keyword

Valid identifiers: x, _private, myFunction, var_123

Invalid identifiers: 123var (starts with digit), my-var (contains hyphen), if (reserved keyword)

4.3 Literals

Literals represent constant values in the program:

Numeric Literals: Decimal integers and floating-point numbers. Examples: 0, 42, -17, 3.14, 2.5e-3

Character Literals: A single character enclosed in single quotes. Examples: 'A', '0', ' ' (space)

String Literals: Sequences of characters enclosed in double quotes. Supports escape sequences like \n (newline), \t (tab), etc. Examples: "hello", "line1\nline2"

Boolean Literals: true and false

5 Data Types

5.1 Primitive Types

Em provides the following primitive data types:

Type Description Size/Range
void No type; used for functions that don't return a value N/A
bool Boolean value (true or false) 1 byte
int or s32 Signed 32-bit integer -2,147,483,648 to 2,147,483,647
s64 Signed 64-bit integer -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807
u32 Unsigned 32-bit integer 0 to 4,294,967,295
u64 Unsigned 64-bit integer 0 to 18,446,744,073,709,551,615
float 32-bit floating-point number IEEE 754 single precision
char Single character 8-bit
string String of characters Variable length

6 Keywords

The following words are reserved keywords in Em and cannot be used as identifiers:

Category Keywords
Control Flow if, else, for, while
Jump Statements return, break, continue
Special Builtins varg

7.2 Operator Precedence and Associativity

Operators are evaluated according to their precedence level. Operators with higher precedence are evaluated first. Operators at the same precedence level are evaluated according to their associativity (left-to-right or right-to-left).

The following table lists operators in order from highest to lowest precedence:

Precedence Level Operators Associativity Description
1 (Highest) ++ -- () [] Left-to-Right Postfix increment/decrement, function calls, array access
2 ++ -- ! ~ + - & * Right-to-Left Prefix unary operators
3 * / % Left-to-Right Multiplicative operators
4 + - Left-to-Right Additive operators
5 << >> Left-to-Right Bitwise shift operators
6 < <= > >= Left-to-Right Comparison operators
7 == != Left-to-Right Equality operators
8 & Left-to-Right Bitwise AND
9 ^ Left-to-Right Bitwise XOR
10 | Left-to-Right Bitwise OR
11 && Left-to-Right Logical AND
12 || Left-to-Right Logical OR
13 (Lowest) = += -= *= /= %= &= |= ^= <<= >>= &&= ||= Right-to-Left Assignment operators

8 Expressions and Statements

8.1 Variable Declarations

Variables are declared using a data type followed by an identifier: 

int x;
float pi = 3.14;
bool flag;
string message = "Hello";

Variables can be initialized at declaration time with a value. If not initialized, the variable contains an undefined value.

Syntax: 

<data_type> <identifier>;
<data_type> <identifier> = <expression>;

8.2 Identifiers

An identifier refers to a variable or function that has been previously declared. Using an identifier retrieves the value stored in that variable or refers to the function. 

int x = 10;
int y = x;  // y gets the value of x

8.3 Literals

Literals are constant values written directly in the code: 

int a = 42;           // numeric literal
float b = 3.14;       // floating-point literal
char c = 'A';         // character literal
string s = "hello";   // string literal
bool flag = true;     // boolean literal

8.4 Unary Expressions

Unary expressions apply an operator to a single operand. Em supports both prefix and postfix unary operators:

Prefix Unary Operators

int x = 5;
int y = -x;       // negation: y = -5
bool flag = !true; // logical NOT: flag = false
int a = ~7;       // bitwise NOT
int b = ++x;      // prefix increment: x is incremented, then value is used
int c = --x;      // prefix decrement: x is decremented, then value is used
int* ptr = &x;    // address-of operator
int val = *ptr;   // dereference operator

Postfix Unary Operators

int x = 5;
int y = x++;      // postfix increment: value is used, then x is incremented
int z = x--;      // postfix decrement: value is used, then x is decremented

8.5 Binary Expressions

Binary expressions combine two operands with a binary operator. The operator's precedence and associativity determine the order of evaluation: 

int a = 5 + 3;      // addition
int b = 10 - 4;     // subtraction
int c = 6 * 2;      // multiplication
int d = 15 / 3;     // division
int e = 17 % 5;     // modulo
bool f = a < b;    // comparison
bool g = a == 5;    // equality
int h = a << 2;   // left shift
int i = a >> 1;   // right shift
int j = a & b;    // bitwise AND
int k = a | b;      // bitwise OR
int l = a ^ b;      // bitwise XOR
bool m = a && b;  // logical AND
bool n = a || b;    // logical OR

8.6 Assignments

Assignment expressions assign a value to a variable. The basic assignment operator is =, and there are also compound assignment operators: 

int x = 10;
x = 20;           // simple assignment
x += 5;           // equivalent to x = x + 5
x -= 3;           // equivalent to x = x - 3
x *= 2;           // equivalent to x = x * 2
x /= 4;           // equivalent to x = x / 4
x %= 3;           // equivalent to x = x % 3
x <<= 1;         // equivalent to x = x << 1
x >>= 1;         // equivalent to x = x >> 1
x &= 0xFF;       // equivalent to x = x & 0xFF
x |= 0x0F;        // equivalent to x = x | 0x0F
x ^= 0xAA;        // equivalent to x = x ^ 0xAA
bool flag = true;
flag &&= false;  // equivalent to flag = flag && false
flag ||= true;    // equivalent to flag = flag || true

8.7 Function Calls

A function call invokes a previously defined function with arguments: 

// Function definition
int add(int a, int b) {
    return a + b;
}

// Function calls
int result = add(3, 5);      // result = 8
int sum = add(result, 10);   // sum = 18

// Function with no parameters
int getValue() {
    return 42;
}

int value = getValue();      // value = 42

// Function with no return value
void printValue(int x) {
    // printing logic would go here
}

printValue(42);

Syntax: 

<function_name>(<arg1>, <arg2>, ...)

8.8 If-Else Statements

Conditional execution using if-else statements: 

int x = 10;

if (x > 5) {
    // executed if x is greater than 5
}

if (x == 10) {
    // executed if x equals 10
} else {
    // executed if x does not equal 10
}

if (x < 0) {
    // x is negative
} else if (x == 0) {
    // x is zero
} else {
    // x is positive
}

The condition must be a boolean expression. If the condition is true, the code block after if is executed. If the condition is false and an else block is present, that block is executed.

8.9 For Loops

A for loop repeats a block of code a specified number of times: 

// Basic for loop
for (int i = 0; i < 10; i++) {
    // this block executes 10 times (i goes from 0 to 9)
}

// For loop with external variable
int i;
for (i = 0; i < 5; i++) {
    // executes 5 times
}

// For loop with break
for (int i = 0; i < 10; i++) {
    if (i == 5) {
        break;  // exits the loop when i equals 5
    }
}

// For loop with continue
for (int i = 0; i < 5; i++) {
    if (i == 2) {
        continue;  // skips to the next iteration
    }
}

Syntax: 

for (<initialization>; <condition>; <update>) {
    <body>
}

The for loop has three parts:

  • initialization: Executed once before the loop starts
  • condition: Checked before each iteration; if false, the loop ends
  • update: Executed after each iteration

8.10 While Loops

A while loop repeats a block of code as long as a condition is true: 

int i = 0;
while (i < 10) {
    // this block repeats while i is less than 10
    i++;
}

// While loop with break
int x = 0;
while (true) {
    x++;
    if (x == 10) {
        break;  // exits the loop
    }
}

// While loop with continue
int y = 0;
while (y < 5) {
    y++;
    if (y == 2) {
        continue;  // skips the rest of this iteration
    }
}

Syntax: 

while (<condition>) {
    <body>
}

The while loop continues executing the body block as long as the condition is true. If the condition is initially false, the body never executes.

8.11 Break and Continue Statements

break and continue are jump statements used within loops:

break: Immediately exits the nearest enclosing loop. 

for (int i = 0; i < 10; i++) {
    if (i == 5) {
        break;  // exits the for loop
    }
}

continue: Skips the rest of the current iteration and jumps to the next iteration. 

for (int i = 0; i < 10; i++) {
    if (i %% 2 == 0) {
        continue;  // skips even numbers
    }
    // process odd numbers
}

8.12 Return Statements

A return statement exits a function and optionally returns a value: 

int add(int a, int b) {
    return a + b;  // returns the sum
}

void printMessage(string msg) {
    // some code here
    return;  // can return without a value in void functions
}

int getValue() {
    if (someCondition) {
        return 10;
    } else {
        return 20;
    }
}

Syntax: 

return;           // for void functions
return <value>;  // returns a value

The return type of the value must match the function's declared return type.

8.13 Function Definitions

A function definition declares a function that can be called elsewhere in the program: 

// Simple function
int add(int a, int b) {
    return a + b;
}

// Function with no parameters
int getConstant() {
    return 42;
}

// Function with no return value
void printNumber(int num) {
    // printing logic
}

// Function with multiple statements
float calculateAverage(float a, float b, float c) {
    float sum = a + b + c;
    float average = sum / 3;
    return average;
}

// Recursive function
int factorial(int n) {
    if (n <= 1) {
        return 1;
    }
    return n * factorial(n - 1);
}

Syntax: 

<return_type> <function_name>(<param1_type> <param1_name>, <param2_type> <param2_name>, ...) {
    <function_body>
}

A function definition includes:

  • return_type: The type of value the function returns
  • function_name: The name of the function
  • parameters: A list of parameter declarations
  • function_body: The code that executes when the function is called

8.14 Function Prototypes

A function prototype declares a function without implementing its body. This is useful for forward declarations and for declaring functions that will be defined elsewhere (e.g., in another file or in a header): 

// Function prototype
int add(int a, int b);

// Later in the same file or in another file:
int add(int a, int b) {
    return a + b;
}

// Function prototype with no parameters
void printMessage();

void printMessage() {
    // implementation
}

Syntax: 

<return_type> <function_name>(<param_types>);

Function prototypes allow you to use a function before its full definition appears in the code, or to declare interfaces for functions defined elsewhere.

8.15 Variadic Arguments

Functions can accept a variable number of arguments using the varg keyword to access them: 

int sum(int count, ...) {
    int total = 0;
    int i = 0;
    while (i < count) {
        total += varg(int);
        i++;
    }
    return total;
}

// Called with varying numbers of arguments
int result1 = sum(2, 5, 10);      // sum of 5 and 10
int result2 = sum(3, 1, 2, 3);    // sum of 1, 2, and 3

Syntax: 

<return_type> <function_name>(<fixed_params>, ...) {
    // use varg(<type>) to retrieve variadic arguments
}

The varg keyword is used within the function to retrieve each variadic argument. The function must know or determine how many variadic arguments were passed.

8.16 Block Expressions

Em supports scoped block expressions that create a new scope for variables: 

int x = 10;

{
    // new scope
    int x = 20;  // shadow the outer x
    int y = 30;
}

// x is still 10 here
// y is not accessible

Block expressions are useful for limiting variable scope and avoiding naming conflicts.

9 Preprocessor Directives

Em supports preprocessor directives that are processed before compilation. These directives allow conditional compilation, file inclusion, and macro definitions.

9.1 #include

The #include directive includes code from another file. There are two forms:

Local includes: 

#include "myheader.emh"

Includes the specified file from the current directory.

Standard library includes: 

#include <standardlib.emh>

Includes a file from the Em standard library. Associated compiled library files (.bc) are automatically linked.

9.2 #define

The #define directive defines a macro that will be replaced wherever it appears: 

#define MAX_SIZE 100
#define PI 3.14159
#define DEBUG_MODE 1

int array[MAX_SIZE];  // MAX_SIZE is replaced with 100

Syntax: 

#define <TOKEN_NAME> <REPLACEMENT>

The token name must start with a letter or underscore. The replacement can be a number, identifier, or other token.

9.3 #undef

The #undef directive undefines a previously defined macro: 

#define FEATURE 1
// ... code using FEATURE ...
#undef FEATURE
// FEATURE is no longer defined

Syntax: 

#undef <TOKEN_NAME>

9.4 #ifdef

The #ifdef directive includes code only if a specific macro is defined: 

#define DEBUG

#ifdef DEBUG
    // this code only compiles if DEBUG is defined
    int flag = 1;
#endif

Syntax: 

#ifdef <MACRO_NAME>
    <code to include if macro is defined>
#endif

9.5 #ifndef

The #ifndef directive includes code only if a specific macro is NOT defined: 

#ifndef PRODUCTION
    // this code compiles in development but not in production
    int debugFlag = 1;
#endif

Syntax: 

#ifndef <MACRO_NAME>
    <code to include if macro is NOT defined>
#endif

9.6 #endif

The #endif directive marks the end of a conditional block started by #ifdef or #ifndef: 

#ifdef FEATURE
    // conditional code
#endif
